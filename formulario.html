Todos los formularios en web, manejan metodos para el envio de las diferentes peticiones que realiza el usuario. Debermos recordar que la mayoria de las aplicaciones en web trabajan sobre una arquitectura cliente servidor. Significa que el cliente realiza una peticion (request) y el servidor debe de atender mediante una respuesta (response)
Los tipos son:
GET:

Este metodo se encarga de enviar la informaciónn del formulario por medio de la salida o url estandar del navegador, esto propicia que pueda existir inyecciones de código realizando violaciones a la seguridad del sistema.

POST:

Este método se encarga de enviar la informacion igual por el método estandar pero mediante de un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo del sistema.

Get solo se usa para operaciones de lectura, post sirve para cambiar la info de un recurso por parte de un servidor (Insertar un registro) Put almacena la informacion de un recurso particular, para su actualización. Head regresa el código de respuesta y cualquier cabecera HTTP asociada
